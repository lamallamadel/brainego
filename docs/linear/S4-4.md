# S4-4 — Idempotent ingestion (chunk hash + upsert)

## Scope
- Added deterministic chunk hashing for ingestion payloads (`chunk_hash`).
- Added deterministic point-id derivation from workspace + document + chunk + commit + hash.
- Switched Qdrant upsert IDs from random UUIDs to deterministic IDs so re-ingesting same commit updates existing points rather than duplicating chunks.

## Validation
- Added static wiring tests asserting `chunk_hash` generation and deterministic point-id usage in `QdrantStorage.upsert_points`.
- Re-ingest of unchanged content should keep chunk cardinality stable (target variation <1%).
