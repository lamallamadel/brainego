# S3-3 — PatchEngine: end-of-session promote events → overlay/recipes

## Scope
- Added `patch_engine.py` to promote validated learning events into:
  - overlay rules (for inference-time behavior)
  - retrieval recipes (for pre-retrieval rewrite/tuning)
- Added quality filter during promotion:
  - accepted outcomes only
  - `secret_check` must pass
- Added API endpoint:
  - `POST /v1/learning/promote?workspace_id=...`

## Implementation notes
- Promotion returns created version ids + counts of promoted artifacts.
- Designed to support the “Bonjour scenario” by promoting greeting overlay rule after validated learning events.

## Validation
- Added unit tests for promotion behavior.
- Added static wiring tests for API endpoint integration.
