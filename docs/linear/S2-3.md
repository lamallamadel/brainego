# S2-3 — RecoveryPlanner (execute candidate_queries + re-score ESS)

## Scope
- Added `recovery_planner.py` to execute teacher-proposed candidate queries and recompute ESS.
- Added config `RECOVERY_MAX_ATTEMPTS` (default 2).
- Integrated recovery loop in `/v1/rag/query` missing-context flow:
  - run candidate queries
  - pick best recovered result set by ESS
  - if ESS reaches threshold, continue as grounded response (`grounded_after_recovery`)
  - otherwise keep `missing_context` response.

## Implementation notes
- Added recovery telemetry in `MetricsStore`:
  - `recovery_attempts`
  - `recovery_successes`
  - `recovery_success_rate`
- Recovery status is exposed in retrieval stats:
  - `recovery_attempts_used`
  - `recovered_ess`
  - `recovery_succeeded`
  - `response_mode` (`missing_context` or `grounded_after_recovery`)

## Validation
- Added unit tests for planner behavior and static wiring in API.
