# S2-2 — Teacher JSON Contract (questions/queries/plan/hypotheses)

## Scope
- Added strict teacher output contract in `teacher_contract.py` using Pydantic.
- Contract fields:
  - `clarifying_questions` (<=3)
  - `candidate_queries`
  - `search_plan`
  - `ingestion_suggestions`
  - `hypotheses`
- Integrated contract validation in API teacher usage paths.

## Implementation notes
- If teacher output is invalid, API now ignores raw teacher payload and stores safe marker:
  - `{"invalid_teacher_output": true}`
- Timeout/error remains safe fallback:
  - `{"timeout_or_error": true}`
- Hypotheses are carried under `teacher_guidance` structure only (not promoted as facts).

## Validation
- Added unit tests for valid/invalid contract payloads.
- Added static wiring tests for contract validation integration in API flow.
