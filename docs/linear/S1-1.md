# S1-1 — Hybrid Retrieval (BM25 + vector + RRF)

## Scope
- Added hybrid retrieval composition in `rag_service.py`:
  - vector retrieval candidate generation
  - lexical BM25-lite ranking over retrieved candidates
  - Reciprocal Rank Fusion (RRF) final ranking
- Added env configuration:
  - `HYBRID_RETRIEVAL_ENABLED` (default true)
  - `HYBRID_RRF_K` (default 60)
  - `HYBRID_VECTOR_CANDIDATE_MULTIPLIER` (default 3)
- Applied hybrid retrieval in both standard search and graph-enriched vector retrieval stage.
- Added explicit cross-workspace leakage assertion on retrieval results.

## Implementation notes
- New helper module: `hybrid_retrieval.py` (`bm25_lite_score`, `rank_bm25_lite`, `fuse_rrf`).
- `RAGIngestionService.search_documents` now fuses vector + lexical ranks.
- `search_with_graph_enrichment` now uses same hybrid fusion policy before graph augmentation.

## Validation
- Added unit tests for ranking/fusion helpers.
- Added static wiring tests for hybrid usage and workspace leakage assertion.
