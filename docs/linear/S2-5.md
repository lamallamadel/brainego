# S2-5 — LearningEvents store (teacher_used, trigger, outcome)

## Scope
- Added `learning_events_store.py` for structured learning events with:
  - schema versioning (`version`)
  - workspace scoping (`workspace_id`)
  - timestamped event payload
  - basic redaction of sensitive key-value patterns
- Added retention helper per workspace: `retain_workspace_last_n`.
- Integrated event recording in API flows for:
  - teacher-assisted missing-context paths
  - support-check downgrade paths

## Implementation notes
- New helper `record_learning_event(workspace_id, event)` in API layer.
- Events capture at least:
  - `trigger`
  - `teacher_used`
  - `outcome`

## Validation
